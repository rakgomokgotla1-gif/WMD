Chapter 1
HTML5
OBJ 1 Create Web Pages with HTML
HTML is a markup language that identifies sections of a web page using elements (tags)
<!DOCTYPE html> <html> <head><title>Title</title></head> <body> visible content </body> </html>
<head>Descriptive info — not shown in browser
<body>All visible content the user sees
DoctypeDeclares document rules/version
OBJ 2 Add Style with CSS
CSS separates presentation from content — controls fonts, colors, spacing
/* selector { property: value; } */ p { color: blue; } h1 { border-bottom: 1px solid; }
One CSS file can style an entire website
OBJ 3 Describe the History of HTML
1989Tim Berners-Lee proposes HTML at CERN
1994W3C founded at MIT to standardize the web
HTML originBased on SGML — Standard Generalized Markup Language
W3C roleSets jointly developed standards benefiting everyone
OBJ 4 Work with HTML5
Element = opening tag + content + closing tag
Void element = single tag only (inserts content)
<br> <img> <input> <link> <meta>
Attribute = extra info inside an opening tag
<h1 id="title">Main Title</h1>
OBJ 5 Use Good Coding Practices
Browser's rendering engine interprets HTML/CSS — different browsers may display differently
Always test in multiple browsers — Chrome, Firefox, Safari, Edge
Validate code; follow W3C standards
HTML markup is hidden from users — only results are shown
Chapter 2
Web Site Design Principles
OBJ 1 Understand the Web Design Environment
Many variables affect how pages appear — most cannot be controlled
Browser type Screen size Connection speed Device/OS Font availability
RuleCode to W3C standards, validate, test on multiple browsers and devices
CacheBrowser stores files locally on return visits — reuse graphics to exploit this
OBJ 2 Design for Multiple Screen Resolutions
Resolution = width × height in pixels. Common: 1024×768, 1366×768
LayoutBehavior
FlexibleAdapts to any screen width
FixedSet width; stays centered
Mobile devices have fixed resolutions — use CSS Media Queries to adapt styles per device
OBJ 3 Craft the Look and Feel of the Site
LookVisual appearance — colors, fonts, layout, images
FeelUser experience — navigation, responsiveness, interactions
Together they define the site's brand identity and overall UX
OBJ 4 Create a Unified Site Design
Consistent style, colors, and navigation across all pages
Reuse CSS and graphic elements for cohesion and faster loading
Design should reflect a clear brand identity throughout
OBJ 5 Design for the User
Know your audience — adopt the user's perspective
Design for varying connection speeds; plan for slow and fast users
Test on multiple devices and operating systems
Users don't like waiting — optimize performance
OBJ 6 Design for Accessibility
ContrastMin 4.5:1 ratio (WCAG) for text readability
KeyboardAll actions operable without a mouse
Alt TextDescriptive text on images for screen readers
W3C WAI (Web Accessibility Initiative) publishes WCAG guidelines

Chapter 4
Cascading Style Sheets
OBJ 1 Recognize the Benefits of Using CSS
Separates presentation from content — structure in HTML, style in CSS
One stylesheet controls the entire site — change once, update everywhere
CSS3 is the latest release; modern browsers offer full support
OBJ 2 Build a Basic Style Sheet
MethodWhereScope
<link> ✅.css fileWhole site
<style><head>One page
style=""elementOne element
/* External link in <head> */ <link href="styles.css" rel="stylesheet">
Write one property per line; use comments for clarity
OBJ 3 Use Inheritance to Write Simpler Style Rules
Child elements inherit styles from parent elements
/* Instead of styling h1, p, ul, em, li... */ body { color: red; } /* all inherit! */
Check property descriptions to know if a property is inherited
OBJ 4–5 Examine & Apply Basic Selection Techniques
Typeh1 { } — targets every matching element
Groupingh1, h2, p { } — one rule for many elements
Combiningh1 { color: red; font-size: 24px; }
Descendantdiv p { } — targets p only inside div
OBJ 6 Use Class and ID Selectors
/* Class — reusable on many elements */ .highlight { background: yellow; } /* ID — unique, one element only */ #header { font-size: 32px; }
Class = reusable style. ID = unique element on the page
OBJ 7 Use the <div> and <span> Elements
<div>Block-level container — groups sections of content
<span>Inline container — styles a portion of text
Paired with class/ID selectors to create layout elements and style hooks
OBJ 8 Use Other Selectors
Attributeimg[title=home]{ } — targets by attribute value
Pseudo-classSelect by state: :link :visited :hover :active
Pseudo-elemp:first-letter p:first-line :before :after
Link pseudo-class order: LoVe HAte — Link, Visited, Hover, Active
Cascade resolves conflicts by: specificity → rule order → !important
