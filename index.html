CH 01

HTML5 — Structure of the Web

Core Concept
HTML is a markup language that identifies and structures sections of a web page using elements (tags). CSS then styles that structure.
Syntax Rules
<!-- Paired element --> <h1>This is a heading</h1> <!-- Void element (no closing tag) --> <br> <img src="img.jpg"> <!-- Attribute syntax --> <h1 id="title">Main Title</h1>
Basic Page Structure
<!DOCTYPE html> <!-- doctype --> <html> <!-- root element --> <head> <!-- meta info --> <title>Page</title> </head> <body> <!-- visible content --> </body> </html>
Key Terms
ElementA pair of HTML tags + their content
Void ElementSingle tag; inserts content (e.g. <br>, <img>)
AttributeExtra info inside a tag (e.g. id, href, src)
DoctypeDeclares the HTML version/rules for the doc
Rendering EngineBrowser component that interprets HTML/CSS
CSS Basics
/* selector { property: value; } */ p { color: blue; font-size: 16px; } h1 { border-bottom: 2px solid red; }
SelectorTargets which element to style
Declarationproperty + value pair inside { }
History Milestones
1989Tim Berners-Lee proposes HTML at CERN
1994W3C founded at MIT to set web standards
HTML5Latest major version; improves media & semantics
CH 02

Web Design Principles

Core Concept
Web pages appear differently across browsers, devices & screen sizes. Designers must plan for every variable they cannot control.
Variables You CANNOT Control
Browser type Screen resolution Connection speed Device type OS / fonts Browser version
Browser Compatibility Guidelines
  • Follow W3C standards — ensures cross-browser consistency
  • Validate your HTML and CSS code
  • Test in both old and new browser versions
  • Know your audience's preferred browser
Screen Resolution & Layout Types
TypeBehaviorWhen to Use
Flexible Stretches/shrinks with screen Modern, responsive sites
Fixed Set width, stays centered Predictable layouts
Common resolutions: 1024×768 and 1366×768
Look & Feel
LookColors, fonts, layout, imagery — the visual design
FeelNavigation, interactions, responsiveness — the UX
Unified DesignConsistent style & brand across all pages
Accessibility Features
Color ContrastMin 4.5:1 ratio (WCAG) for readable text
Keyboard NavAll actions operable without a mouse
Alt TextImage descriptions for screen readers
Browser Cache
On repeat visits, browsers load files from local cache (faster). Reuse graphics across pages to maximize this benefit.
Mobile Design
  • Use CSS Media Queries to apply different styles per device
  • Test on actual smartphones and tablets
  • Mobile devices have fixed screen resolutions
CH 03

Site Planning

Core Concept
Plan before you build. Every successful site starts with a project specification, a clear goal, and a deep understanding of the audience.
Development Lifecycle (In Order)
1Requirements & Specs — Client needs, audience, technical constraints
2Information Design — Taxonomy: classify & organize content hierarchy
3Mockups & Wireframes — Sketch layouts; wireframes = detailed blueprints
4Construction — Code the site once design is mostly finalized
5QA & User Testing — Validate tech AND usability with real users
6Publishing & Promotion — Launch + marketing
7Ongoing Maintenance — Keep content fresh after launch
Site Specification Questions
  • Who is the client and what are the goals?
  • Who is the target audience?
  • What are the technical & budget constraints?
  • How will you measure success?
  • Is this a new site or upgrade?
Types of Web Sites
Billboard Publishing Portal Blog E-commerce Social Network Wiki RSS Virtual Gallery Intranet/Extranet
Audience Analysis
WHO ARE THEY?
  • Gender / age
  • Education level
  • Technical skill
  • Reading level
WHY DO THEY VISIT?
  • Get information?
  • Download files?
  • Find links?
  • Make a purchase?
Key Terms
TaxonomyClassification & hierarchy of site content for navigation
WireframeDetailed stable layout blueprint before coding
Web AnalyticsStats from web server to track user activity & traffic
CH 04

Cascading Style Sheets (CSS)

Core Concept
CSS separates presentation from content. One style sheet can control the entire site's visual design.
Anatomy of a Style Rule
/* selector { property : value ; } */ p { color : blue ; } h1 { font-size: 24px ; }
SelectorWhich element(s) to target
PropertyWhat characteristic to change (color, font-size…)
ValueThe specific setting for that property
3 Ways to Apply CSS
MethodLocationScope
External ✅ Separate .css file, linked via <link> Whole site
Internal <style> in <head> One page
Inline style="" attribute on element One element
Linking External CSS
<head> <link href="styles.css" rel="stylesheet" type="text/css"> </head>
Inheritance
Child elements inherit styles from parents. Set body { color: red; } and all text turns red — no need to style every element separately.
/* Without inheritance: 5 rules */ h1,p,ul,em,li { color: red; } /* With inheritance: 1 rule */ body { color: red; }
Selector Types
Typeh1 { } — targets ALL matching elements
Groupingh1, h2, p { } — multiple elements, one rule
Class.highlight { } — reusable on many elements
ID#header { } — unique, one element only
Descendantdiv p { } — targets p inside div only
Clean CSS Code Tips
  • One property per line for readability
  • Use comments /* like this */ to explain blocks
  • Use <div> (block) and <span> (inline) as hooks for styling
  • External CSS is best practice for maintainability

TESTING

Website Testing Variables

1. Browser Test on Chrome, Firefox, Safari, Edge — each renders differently
2. Responsive Check layout on desktop, tablet, and mobile screen sizes
3. Performance Measure page load speed; optimize images and resources
4. Security Check for SQL injection, auth flaws, and data vulnerabilities
